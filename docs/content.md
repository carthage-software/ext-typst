# ext-typst

A PHP extension that embeds the [Typst](https://typst.app/) typesetting engine, giving you fast, programmable document generation (PDF, PNG, JPEG, SVG) directly from PHP.

Built with Rust. No CLI needed. No subprocess spawning.

---

## Installation

### Via PIE (PHP Installer for Extensions)

```bash
pie install carthage-software/ext-typst
```

### Pre-built shared libraries

Download the appropriate ZIP archive from the [GitHub Releases](https://github.com/carthage-software/ext-typst/releases) page. Release assets follow the naming convention:

```
php_typst-{version}_php{php version}-{architecture}-{platform}-{libc}[-zts].zip
```

NTS (Non-Thread Safe) builds have no suffix. ZTS (Thread Safe) builds end with `-zts`.

For example: `php_typst-0.1.0_php8.4-x86_64-linux-glibc.zip`, `php_typst-0.1.0_php8.3-arm64-darwin-bsdlibc-zts.zip`.

Each ZIP contains a `typst.so` file. After extracting, load it in your `php.ini`:

```ini
extension=/path/to/typst.so
```

### Supported platforms

| Platform | Architecture |
| -------- | ------------ |
| Linux    | x86_64       |
| Linux    | aarch64      |
| macOS    | arm64        |

Both NTS (Non-Thread Safe) and ZTS (Thread Safe) builds are available for PHP 8.3, 8.4, and 8.5.

### IDE and static analysis support

Install the Composer package to get PHP stubs for autocompletion and static analysis:

```bash
composer require --dev carthage-software/ext-typst
```

This enables full support in IDEs like PhpStorm and static analyzers like Mago, PHPStan, and Psalm.

### Verify installation

```bash
php -m | grep typst
```

## Quick Start

```php
<?php

$world = new Typst\World();
$compiler = new Typst\Compiler($world);

$document = $compiler->compileString(<<<'TYPST'
    #set page(height: auto)

    = Hello from PHP

    This PDF was generated by *Typst* inside a PHP extension.

    - No subprocess spawning
    - No temporary files
    - No CLI needed
    TYPST);

$document->toPdf()->save('hello.pdf');
```

That's it. A PDF, generated in-process, in milliseconds.

## World

The `Typst\World` class is the compilation environment. It manages fonts, source loading, caching, and configuration. It is shared across `Compiler` and `Inspector` instances.

### Creating a world

```php
// Default settings (embedded fonts, current working directory)
$world = new Typst\World();

// With options
$world = new Typst\World(
    template_dir: __DIR__ . '/templates',
    font_dirs: ['/usr/share/fonts'],
    embed_default_fonts: true,
    package_dir: '/tmp/typst-packages',
    cache_size: 64,
);
```

| Parameter             | Type      | Default | Description                                   |
| --------------------- | --------- | ------- | --------------------------------------------- |
| `template_dir`        | `?string` | `null`  | Root directory for resolving relative file paths and imports |
| `cache_size`          | `?int`    | `64`    | Maximum cached file/source entries (0 = none)  |
| `embed_default_fonts` | `?bool`   | `true`  | Include bundled default fonts                  |
| `font_dirs`           | `?array`  | `null`  | Directories to scan for font files             |
| `package_dir`         | `?string` | `null`  | Local directory for Typst packages             |

### Loading sources

```php
// From a string
$source = $world->loadString('#set page(height: auto)\nHello World');

// From a file (relative to template_dir)
$source = $world->loadFile('report.typ');

// From an absolute path
$source = $world->loadFile('/absolute/path/to/document.typ');
```

Relative paths in `loadFile()` are resolved against the `template_dir`. The returned `Typst\Source` object can be compiled multiple times with different inputs.

### Listing available fonts

```php
$families = $world->getFontFamilies();
// e.g. ['DejaVu Sans', 'New Computer Modern', 'Roboto', ...]
```

### Cloning

Use `clone` to create an independent copy that shares base fonts but has its own instance fonts. Sources created by the original world (or any of its clones) remain compatible:

```php
$world2 = clone $world;
$world2->addFontFile('/path/to/extra-font.ttf'); // only affects $world2

// Sources from either world work with compilers from either
$source = $world->loadString('Hello');
$compiler = new Typst\Compiler($world2);
$compiler->compile($source); // works - same world lineage
```

### Clearing caches

In long-lived processes (e.g. RoadRunner), you can clear internal file/source caches to free memory or force re-reading changed files:

```php
$cleared = $compiler->clearCache(); // returns number of cleared entries
```

Both `Compiler` and `Inspector` have independent caches and both expose `clearCache()`.

## Compiler

The `Typst\Compiler` class compiles Typst sources into documents. It throws on compilation failure. Compilation warnings are silently discarded; use `Inspector` if you need access to warnings.

```php
$world = new Typst\World(template_dir: __DIR__ . '/templates');
$compiler = new Typst\Compiler($world);

// Compile a Source object
$source = $world->loadString('Hello');
$document = $compiler->compile($source);

// Convenience: compile from string directly
$document = $compiler->compileString('Hello');

// Convenience: compile from file (relative to template_dir)
$document = $compiler->compileFile('report.typ');

// Access the world
$world = $compiler->getWorld();
```

### Passing input variables

Input variables are accessible in Typst via `sys.inputs`. Values can be strings, integers, floats, booleans, null, or nested arrays:

```php
$document = $compiler->compileString(
    '#sys.inputs.at("name")',
    ['name' => 'Azjezz', 'count' => 42, 'active' => true],
);
```

## Inspector

The `Typst\Inspector` class compiles Typst sources and returns diagnostics instead of throwing. Use it when you want to handle errors and warnings programmatically.

```php
$world = new Typst\World();
$inspector = new Typst\Inspector($world);

$result = $inspector->inspectString('#unknown()');

if (!$result->success()) {
    foreach ($result->errors() as $error) {
        echo "Error: {$error->message()}\n";

        if ($span = $error->span()) {
            echo "  at {$span->file()}:{$span->line()}:{$span->column()}\n";
        }

        foreach ($error->hints() as $hint) {
            echo "  hint: {$hint}\n";
        }
    }
}

// On success, get the document
if ($result->success()) {
    $document = $result->getDocument();
}
```

The inspector has the same convenience methods as the compiler:

```php
$result = $inspector->inspect($source);        // Source object
$result = $inspector->inspectString($string);  // string
$result = $inspector->inspectFile($path);      // file path
```

## Source

A `Typst\Source` represents a loaded Typst source. It is created by `World::loadString()` or `World::loadFile()` and can be compiled multiple times with different inputs.

A source is bound to the world that created it. Passing a source from one world to a compiler created from a different world throws `InvalidArgumentException`. Cloned worlds share the same lineage, so sources remain compatible across clones.

```php
$source = $world->loadString('Hello #sys.inputs.at("name")');

// Compile with different inputs
$doc1 = $compiler->compile($source, ['name' => 'Alice']);
$doc2 = $compiler->compile($source, ['name' => 'Bob']);

// Inspect the source
echo $source->getId();   // opaque identifier
echo $source->getText(); // source text content
```

## Document

A compiled `Typst\Document` can be exported to PDF, images, or SVG.

```php
$document = $compiler->compileString('= Hello World');

echo $document->pageCount(); // Number of pages

// Page dimensions in typographic points (1 pt = 1/72 inch)
echo $document->pageWidth();       // First page width
echo $document->pageHeight();      // First page height
echo $document->pageWidth(page: 2); // Third page width

// Export to different formats
$pdf = $document->toPdf();
$image = $document->toImage();           // First page, PNG, 144 DPI
$svg = $document->toSvg();              // First page

// Specific page (zero-indexed)
$image = $document->toImage(page: 2);    // Third page
$svg = $document->toSvg(page: 0);       // First page

// All pages
$images = $document->toImages();         // array of Image objects
$svgs = $document->toSvgs();            // array of Svg objects
```

## Image Options

Control format, quality, and resolution for image exports:

```php
$options = new Typst\ImageOptions(
    format: Typst\ImageFormat::Jpeg,
    quality: 90,    // 1-100, JPEG only
    dpi: 300.0,     // dots per inch
);

// Or use the builder pattern
$options = (new Typst\ImageOptions())
    ->withFormat(Typst\ImageFormat::Png)
    ->withDpi(300.0);

$image = $document->toImage(options: $options);
```

| Property  | Type          | Default | Description                               |
| --------- | ------------- | ------- | ----------------------------------------- |
| `format`  | `ImageFormat` | `Png`   | `ImageFormat::Png` or `ImageFormat::Jpeg` |
| `quality` | `int`         | `85`    | JPEG quality (1-100)                      |
| `dpi`     | `float`       | `144.0` | Resolution in dots per inch               |

## PDF Options

Control identifier, timestamp, page range, standard conformance, and tagging for PDF exports:

```php
$options = new Typst\PdfOptions(
    identifier: 'my-document',
    timestamp: time(),
    first_page: 0,    // zero-based
    last_page: 4,     // zero-based, inclusive
    version: Typst\PdfVersion::V17,
    validator: Typst\PdfValidator::A2B,
    tagged: true,     // accessibility tagging
);

// Or use the builder pattern
$options = (new Typst\PdfOptions())
    ->withFirstPage(2)
    ->withLastPage(10)
    ->withTagged(false);

$pdf = $document->toPdf($options);
```

| Property     | Type            | Default | Description                                    |
| ------------ | --------------- | ------- | ---------------------------------------------- |
| `identifier` | `?string`       | `null`  | Document identifier for PDF ID generation       |
| `timestamp`  | `?int`          | `null`  | Creation timestamp (Unix seconds, UTC)           |
| `firstPage`  | `?int`          | `null`  | Zero-based first page to include                |
| `lastPage`   | `?int`          | `null`  | Zero-based last page to include (inclusive)      |
| `version`    | `?PdfVersion`   | `null`  | PDF version to target (default: 1.7)            |
| `validator`  | `?PdfValidator` | `null`  | PDF compliance validator (PDF/A or PDF/UA)      |
| `tagged`     | `bool`          | `true`  | Tagged PDF for accessibility                    |

Available `PdfVersion` values: `V14`, `V15`, `V16`, `V17`, `V20`.

Available `PdfValidator` values: `A1B`, `A1A`, `A2B`, `A2U`, `A2A`, `A3B`, `A3U`, `A3A`, `A4`, `A4F`, `A4E` (PDF/A archival), `Ua1` (PDF/UA accessibility).

Not all version + validator combinations are compatible. Incompatible combinations throw `InvalidArgumentException`.

## Output

All output types (`Pdf`, `Image`, `Svg`) implement `Typst\Output\OutputInterface`:

```php
// Get raw bytes
$bytes = $pdf->bytes();

// Get size in bytes
$size = $pdf->size();

// Save to file
$pdf->save('/path/to/output.pdf');

// Use as string (Stringable)
echo $pdf; // raw bytes
```

### Chunked reading

For large outputs, read in chunks to avoid copying the entire buffer:

```php
$pdf = $document->toPdf();
$total = $pdf->size();
$chunk = 8192;

$fp = fopen('output.pdf', 'wb');
$offset = 0;
while ($offset < $total) {
    fwrite($fp, $pdf->bytes(offset: $offset, limit: $chunk));
    $offset += $chunk;
}
fclose($fp);
```

Parameters:

| Parameter | Type   | Default | Description                       |
| --------- | ------ | ------- | --------------------------------- |
| `offset`  | `?int` | `null`  | Byte offset to start reading from |
| `limit`   | `?int` | `null`  | Maximum number of bytes to return |

- Negative `offset` or `limit` throws `InvalidArgumentException`
- `offset` beyond size throws `OutOfBoundsException`
- `limit` exceeding remaining bytes clamps to end

### Output-specific methods

**Pdf:**

```php
$pdf->pageCount(); // int
```

**Image:**

```php
$image->format(); // ImageFormat enum
$image->width();  // int (pixels)
$image->height(); // int (pixels)
```

## Background Compilation

The `Compiler` can offload compilation to a background OS thread, keeping the main thread free for other work. This is essential for non-blocking PHP applications (e.g. Amp, ReactPHP, PSL HTTP servers) where a synchronous compile would freeze the event loop.

```php
$pending = $compiler->compileInBackground($source, $inputs);
```

This returns a `Typst\PendingDocument` immediately. The compilation runs on a separate thread. When it finishes, a notification stream becomes readable.

### Integrating with an event loop

The `PendingDocument::getNotificationStream()` method returns a standard PHP stream resource that becomes readable when compilation finishes. Register it with **any** event loop:

The extension has **no dependency** on any specific event loop or async framework. It hands you a file descriptor - what you do with it is up to you.

### Fire-and-forget batch

Launch multiple compilations, do other work, collect results later:

```php
// Launch all compilations
$pending = [];
foreach ($sources as $source) {
    $pending[] = $compiler->compileInBackground($source);
}

// Do other work while threads compile...
processOtherStuff();

// Collect results
$documents = array_map(fn($p) => $p->join(), $pending);
```

### Blocking usage

If you don't need async, `join()` blocks until compilation finishes:

```php
$document = $compiler->compileInBackground($source)->join();
```

This has minimal overhead (~0.3ms) over a synchronous `compile()` call.

### Concurrent compilation with PSL

```php
use Psl\Async;

function compileAsync(Typst\Compiler $compiler, Typst\Source $source): Typst\Document
{
    $pending = $compiler->compileInBackground($source);
    $stream = $pending->getNotificationStream();

    $suspension = Async\Scheduler::getSuspension();
    Async\Scheduler::onReadable($stream, static function (string $id) use ($suspension): void {
        Async\Scheduler::cancel($id);
        $suspension->resume();
    });

    $suspension->suspend();

    return $pending->join();
}

// Compile 4 documents concurrently - each on its own OS thread
$documents = Async\concurrently([
    fn() => compileAsync($compiler, $source1),
    fn() => compileAsync($compiler, $source2),
    fn() => compileAsync($compiler, $source3),
    fn() => compileAsync($compiler, $source4),
]);
```

### PendingDocument API

| Method                     | Returns    | Description                                          |
| -------------------------- | ---------- | ---------------------------------------------------- |
| `isReady()`                | `bool`     | Non-blocking check if compilation finished           |
| `getNotificationStream()`  | `resource` | Readable stream that lights up on completion         |
| `join()`                   | `Document` | Blocks until done, returns document, closes pipe     |

- `join()` can only be called once. Subsequent calls throw `LogicException`
- `getNotificationStream()` throws `LogicException` after `join()` has been called
- Dropping a `PendingDocument` without calling `join()` cleans up the thread and pipe

### How it works

When you call `compileInBackground()`:

1. A Unix socket pair is created
2. A new OS thread is spawned with an independent compilation world (shared fonts, fresh caches)
3. The thread compiles the document
4. On completion, a single null byte is written to the socket, making the read end readable
5. `join()` collects the thread result and closes the pipe

No Zend internals are hooked. No interrupt handlers. No polling. Just a file descriptor and a thread.

## Error Handling

All exceptions implement `Typst\Exception\ExceptionInterface` (extends `Throwable`).

```php
use Typst\Exception\RuntimeException;
use Typst\Exception\InvalidArgumentException;
use Typst\Exception\OutOfBoundsException;
use Typst\Exception\LogicException;

try {
    $document = $compiler->compileString($source);
} catch (RuntimeException $e) {
    match ($e->getCode()) {
        RuntimeException::COMPILATION_FAILED => handleCompilationError($e),
        RuntimeException::FILE_NOT_FOUND => handleFileNotFound($e),
        RuntimeException::WRITE_FAILED => handleWriteError($e),
        RuntimeException::FONT_INVALID => handleFontError($e),
        RuntimeException::ENCODING_FAILED => handleEncodingError($e),
    };
}
```

### Exception classes

| Class                      | Extends                     | When                                            |
| -------------------------- | --------------------------- | ----------------------------------------------- |
| `RuntimeException`         | `\RuntimeException`         | Compilation failures, I/O errors, font errors   |
| `InvalidArgumentException` | `\InvalidArgumentException` | Invalid quality, DPI, negative offsets           |
| `OutOfBoundsException`     | `\OutOfBoundsException`     | Page index or byte offset beyond range           |
| `LogicException`           | `\LogicException`           | Calling `join()` twice, accessing resolved pending   |

### RuntimeException codes

| Constant             | Value | Description              |
| -------------------- | ----- | ------------------------ |
| `COMPILATION_FAILED` | `1`   | Typst compilation failed |
| `FILE_NOT_FOUND`     | `2`   | File not found           |
| `WRITE_FAILED`       | `3`   | File write failed        |
| `FONT_INVALID`       | `4`   | Invalid font data        |
| `ENCODING_FAILED`    | `5`   | Encoding failed          |

## Diagnostics

The `Inspector` class returns a `Typst\Diagnostic\CompilationResult` instead of throwing.

```php
$inspector = new Typst\Inspector($world);
$result = $inspector->inspectString($source);

// Check outcome
$result->success();      // bool
$result->hasErrors();    // bool
$result->hasWarnings();  // bool

// Get diagnostics
$result->diagnostics();  // all
$result->errors();       // errors only
$result->warnings();     // warnings only

// Each diagnostic
foreach ($result->diagnostics() as $diag) {
    $diag->severity();   // Severity::Error or Severity::Warning
    $diag->message();    // string
    $diag->span();       // ?SourceSpan
    $diag->hints();      // string[]
    echo $diag;          // "error: message (at file:line:column)"
}

// Extract the document (can be called multiple times)
$document = $result->getDocument(); // ?Document (null if failed)
```
